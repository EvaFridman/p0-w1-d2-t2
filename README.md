# p0-w1-d2-t2 · Форма регистрации, которая не бесит

> Тип: новый материал · Репозиторий: один отдельный

## Контекст

Почти в каждом продукте есть экран регистрации, и почти каждый сделан так, что хочется
закрыть вкладку. Твоя задача — собрать форму, которую приятно заполнять: понятные подписи,
правильные типы полей, нативная проверка. Перед стартом посмотри 2–3 хорошие формы в реальных
сервисах — отметь, что в них удобно.

```
форма (method=GET, action=эхо-сервис)
┌──────────────────────────────────────────────┐
│ email · пароль · ник                         │
│ дата · аватар(file) · о себе · тариф         │
└──────────────────────────────────────────────┘
        │ submit (без JS)
        ▼
данные уходят в URL ?email=...&nick=...
сервис показывает, что получил
```

## Что нужно сделать

Форма регистрации на чистом HTML (без CSS-красоты и без JS — его ещё не проходили): email,
пароль, ник, дата рождения, аватар (`file`), короткое «о себе» (`textarea`), выбор тарифа
(`select`). Все поля связаны с подписями через `label`/`for`. Отправка — нативная, через
атрибуты формы `method` и `action`: ставишь `method="GET"` и `action` на любой публичный
эхо-сервис форм (например `https://httpbin.org/get`), жмёшь submit — и видишь, что собралось:
имена полей и значения уедут в query-строку URL, а сервис покажет их в ответе. **Поля «пол»
не делать.**

## Релизы

### Release 1 — поля и подписи
Свёрстать все поля, каждое с `label for`. Поставить осмысленные `type` (email→`email`,
дата→`date`). Кнопка submit. Закоммитить.

### Release 2 — нативная отправка
Прописать форме `action` (эхо-сервис) и `method="GET"`. Проверить `name` у каждого поля —
именно по нему данные попадают в запрос. Нажать submit, найти свои значения в URL и в ответе
сервиса. В `NOTES.md` записать, что делают `name`, `action`, `method` и чем `GET` отличается
от `POST` при отправке.

### Release 3 — приятность
Добавить `placeholder`, `autocomplete`, нужные `required`. Сгруппировать связанные поля.
В `NOTES.md` дописать, что из подсмотренных хороших форм ты применила и почему.

## Критерии приёмки

* Каждое поле связано с `label` через `for`/`id` (клик по подписи фокусирует поле).
* У каждого поля есть осмысленный `name`; после submit значения видны в query-строке URL.
* Типы полей соответствуют смыслу (email — `email`, дата — `date`, аватар — `file`).
* В `NOTES.md` своими словами объяснены `name`/`action`/`method` и разница `GET`/`POST`.
* Поля «пол» нет. Ни строки JavaScript.
